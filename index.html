Erlang

by Josh Rendek

Erlang & OTP - Open Telecom Platform

  • 1987-1991
    • Several design phases worked through including the first Erlang VM's and compilers
  • 1990
    • Erlang is presented at the 1990 International Switching Symposium
  • 1991-1993
    • Erlang is released to the general public
    • Seperate team at Ericsson takes over maintenance and support of the language
  • 1998 Ericsson announces AXD301, telecom switch; over a million lines of Erlang
    • 99.999999999% availability: 1 second of downtime in 30 years

The Erlang Virtual Machine (BEAM)

  • Erlang only supports compiled modules for items such as functions and modules (can't be defined in REPL shell)
    • There is another way using inline function declaration.
    • Not great for large functions.
  • Variables must start with an uppercase letter
    • May only be bound once
  • To evaluate the statement must end with a .
$ erl> io:write('Hello World\n').
        'Hello World\n'ok
$ erl> io:format('Hello World\n', []).
        Hello World
        ok

Errors

4 types of errors

  • Compile-time errors
  • Logical errors
  • Run-time errors
  • Generated errors

Compile-time errors

  • Caught by the compiler, not a big deal

Logical Errors

  • Things like button clicks not triggering events
  • Program does not crash

Generated Errors

  • Will call exit/1 or throw/1
  • Exceptions of classes exit and throw

Run-Time errors

  • Trying to use two different types together ( ex: Lists & Tuples )
  • Process execution stops at this point
  • Emits an exit signal and reason
    • exit reason is a tuple: {Reason, Stack}

Handling of Run-Time errors in Erlang

  • Can prevent process termination using try/catch
  • Can detect process termination through other processes

Processes

  • Allows for massive concurrency
  • Cheap to spawn, low footprint
  • Low scheduling overhead
  • Created by calling spawn
  • Can communicate using send and receive
